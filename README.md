# Task Management System (SSR)

A fully responsive task management application, designed and developed using a **Server-Side Rendering (SSR)** architecture for optimized performance and SEO.

## 🌐 Overview

This project is built with a unified server-driven approach:
* **Server-Side Rendering (SSR)** – The frontend is dynamically generated on the server before being sent to the client, ensuring faster initial load times.
* **Backend Core** – Developed with **Node.js** and **Express**, managing both the view engine and the application logic.
* **Styling** – Implemented with custom **CSS** to provide a professional, desktop-grade task management interface.

The system was designed to centralize logic on the server, providing a robust and secure way to handle task data and user interactions.

## 🧩 Features

* **SSR Architecture** – Server-rendered views for improved data consistency and initial page load speed.
* **Task CRUD Operations** – Full lifecycle management (Create, Read, Update, Delete) for tasks.
* **Modular Routing** – Organized server-side routes for view rendering and API logic.
* **Middleware Integration** – Custom layers for request processing, validation, and session management.
* **Fully Responsive Design** – Responsive layouts that adapt to various screen sizes despite the server-side focus.

## 💻 Tech Stack

### Frontend & Rendering
* **SSR (Server-Side Rendering)**
* **Template Engine** (e.g., EJS/Pug/Handlebars)
* **CSS3** & **JavaScript**

### Backend
* **Node.js**
* **Express.js**
* **Middleware** (Custom implementation)
